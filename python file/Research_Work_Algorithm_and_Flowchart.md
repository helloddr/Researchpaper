# Research Workflow Flowchart Notes

This version is a true branching flowchart for the research work:

- one branch for the main `cardio_train.csv` modeling pipeline
- one branch for the external `UCI Heart Disease` validation pipeline
- both branches merge into a comparison and final conclusion stage

Canonical saved file:

- `../Methodology/research_work_flowchart.png`

The diagram uses:

- rounded boxes for start and end
- rectangles for process steps
- diamonds for decision points

Main message of the chart:

1. Build and optimize the model on the main dataset.
2. Validate the framework on a different dataset.
3. Compare both outcomes.
4. Conclude robustness and generalizability.
